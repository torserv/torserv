🔐 Welcome to TorServe

TorServe is a zero-config, hardened, static web server that launches automatically as a Tor hidden service. It is built for anonymous publishing, especially in censored or high-risk environments.

Unzip → Start Server → Working Hidden Service In Seconds

All this in seconds, no setup headaches, or the risks using insecure web server software.
Binary package available for Linux, Windows, Raspberry Pi

⚠️ Notice: Early Development

This project is young and actively evolving. Although many security precautions are in place, treat TorServe as experimental software. Do not rely on it alone for high-risk operations. Use with extreme caution in hostile environments.

🛡️ Core Safety Features

  • Tor-only access - no clearnet exposure at all
  • Local-only bind (127.0.0.1)
  • No logging (access, error, request)
  • No analytics or telemetry
  • EXIF metadata scrubbing on startup and dynamically if it is running
  • Header stripping - removes User-Agent, Referer, ETag, etc.
  • Randomized response timing (50–200ms jitter)
  • Key rotation with command line arg --new-key
  • Response padding to normalize transfer sizes
  • Obfuscated URLs and file path structures
  • No directory indexing or auto-listing
  • Server refuses to run if unsafe file types are present in public/ directory
  • Dynamic file checking - won’t serve unsafe file types even if added while running

🧭 How to Use Safely

  1. Use only over Tor Browser. Never access from a regular browser.
  2. Scrub your content of metadata before uploading — even though TorServe removes most metadata, it's safest to sanitize everything yourself.
  3. Avoid personal info in file names, folder names, file content and images.
  4. Do not reveal server structure - links and directories are not indexed or guessable by default.
  5. Use on isolated machines (Tails, Qubes, or virtual machines preferred).
  6. Monitor your network to ensure the server is only reachable via .onion.
  7. Keep updated - follow project updates for patches and new features.

🛡️ 核心安全特性

  • 仅支持 Tor 访问 - 完全无明网暴露
  • 仅绑定本地地址（127.0.0.1）
  • 无任何日志记录（访问、错误、请求）
  • 无分析或遥测行为
  • 启动时和运行中自动清除 EXIF 元数据
  • 移除请求头信息 - User-Agent、Referer、ETag 等
  • 响应时间随机延迟（50–200 毫秒）
  • 命令行参数 --new-key 支持密钥轮换
  • 响应填充 — 正规化传输大小
  • 混淆链接和文件路径结构
  • 禁止目录索引和自动列目录
  • 如果 public/ 目录中存在不安全文件类型，服务器将拒绝运行
  • 动态文件检查 - 即使运行中添加了不安全文件也不会提供服务

🧭 安全使用指南

  1. 仅通过 Tor 浏览器访问，绝不能使用普通浏览器访问
  2. 上传前请手动清除文件的元数据，虽然 TorServ 会清除大部分信息，但自己处理最为安全
  3. 文件名、文件夹名、文件内容和图像中避免包含任何个人信息
  4. 不要暴露服务器结构 - 链接和目录默认不会被索引或猜出
  5. 优先在隔离环境中使用（推荐 Tails、Qubes 或虚拟机）
  6. 监控你的网络，确保服务器仅通过 .onion 地址可访问
  7. 保持更新 - 关注项目进展，及时获取安全补丁和新功能

🔐 Bienvenido a TorServ

TorServ es un servidor web estático, endurecido y sin configuración previa que se inicia automáticamente como un servicio oculto de Tor. Está diseñado para la publicación anónima, especialmente en entornos censurados o de alto riesgo.

Descomprimir → Iniciar servidor → Servicio oculto funcional en segundos

Todo esto en segundos, sin complicaciones de configuración ni los riesgos de usar software de servidor web inseguro.
Paquete binario disponible para Linux, Windows, Raspberry Pi

⚠️ Aviso: Desarrollo Temprano

Este proyecto es joven y está en constante evolución. Aunque ya incorpora varias medidas de seguridad, considera TorServ como software experimental. No confíes exclusivamente en él para operaciones de alto riesgo. Úsalo con extrema precaución en entornos hostiles.

🛡️ Funciones de Seguridad Principales

  • Acceso exclusivo por Tor - sin exposición a la red abierta
  • Solo escucha en localhost (127.0.0.1)
  • Sin registros (acceso, errores o solicitudes)
  • Sin analítica ni telemetría
  • Eliminación de metadatos EXIF al iniciar y durante la ejecución
  • Eliminación de cabeceras - User-Agent, Referer, ETag, etc.
  • Retardo de respuesta aleatorio (50–200 ms)
  • Rotación de claves con el argumento --new-key
  • Relleno de respuestas para normalizar tamaños de transferencia
  • Enlaces y rutas de archivos ofuscados
  • Sin indexación ni listado automático de directorios
  • El servidor se niega a iniciar si hay archivos inseguros en el directorio public/
  • Verificación dinámica - el servidor no sirve archivos inseguros aunque se añadan en tiempo de ejecución

🧭 Cómo Usarlo de Forma Segura

  1. Usa solo a través del Navegador Tor. Nunca accedas con un navegador normal.
  2. Elimina los metadatos de tu contenido antes de subirlo — aunque TorServ borra la mayoría, lo más seguro es hacerlo tú mismo.
  3. Evita incluir información personal en nombres de archivos, carpetas, contenido o imágenes.
  4. No reveles la estructura del servidor - los enlaces y directorios no se indexan ni son adivinables por defecto.
  5. Úsalo en sistemas aislados (se recomienda Tails, Qubes o máquinas virtuales).
  6. Supervisa tu red para asegurarte de que el servidor solo sea accesible por .onion
  7. Mantente actualizado - sigue el proyecto para recibir parches y nuevas funciones.

🔐 خوش آمدید به TorServ

TorServ یک سرور وب ایستا، مقاوم‌سازی‌شده و بدون نیاز به پیکربندی است که به‌طور خودکار به‌عنوان یک سرویس مخفی Tor اجرا می‌شود. این ابزار برای انتشار ناشناس طراحی شده است، به‌ویژه در محیط‌های سانسورشده یا پرخطر.

از حالت فشرده خارج کن → سرور را اجرا کن → سرویس مخفی فعال در چند ثانیه

همه این‌ها فقط در چند ثانیه، بدون دردسرهای پیکربندی یا ریسک‌های استفاده از نرم‌افزارهای ناامن سرور وب.
نسخه باینری برای لینوکس، ویندوز و رزبری‌پای در دسترس است

⚠️ توجه: در حال توسعه اولیه

این پروژه هنوز نوپا و در حال تکامل مداوم است. با اینکه بسیاری از اقدامات امنیتی پیاده‌سازی شده‌اند، TorServ را نرم‌افزاری آزمایشی در نظر بگیرید. در عملیات‌های پرخطر، تنها به آن اتکا نکنید. در محیط‌های خصمانه با بالاترین احتیاط از آن استفاده نمایید.

🛡️ ویژگی‌های اصلی امنیتی

  • دسترسی فقط از طریق Tor - هیچگونه تماس با clearnet
  • محدود به اتصال محلی (127.0.0.1)
  • بدون لاگ‌برداری (ورود، خطا، درخواست)
  • بدون تحلیل‌گر یا تله‌متری
  • حذف فراداده‌های EXIF هنگام اجرا و به‌صورت پویا
  • حذف هدرهای HTTP - مانند User-Agent، Referer، ETag و ...
  • تأخیر تصادفی در پاسخ‌ها (۵۰ تا ۲۰۰ میلی‌ثانیه)
  • چرخش کلید با آرگومان خط فرمان --new-key
  • پدینگ پاسخ برای یکسان‌سازی حجم انتقال
  • آدرس‌ها و مسیرهای فایل به صورت مخفی و پیچیده
  • عدم فهرست‌بندی یا لیست خودکار دایرکتوری‌ها
  • در صورت وجود فایل ناامن در پوشهٔ public/، سرور اجرا نخواهد شد
  • بررسی پویا - حتی در حال اجرا نیز فایل‌های ناامن ارائه نمی‌شوند

🧭 راهنمای استفاده ایمن

  1. فقط از طریق مرورگر Tor استفاده کنید. هرگز از مرورگر معمولی استفاده نکنید.
  2. قبل از بارگذاری، فراداده‌های محتوای خود را حذف کنید — اگرچه TorServ اغلب اطلاعات را حذف می‌کند، ولی بهترین کار پاک‌سازی دستی است.
  3. از قرار دادن اطلاعات شخصی در نام فایل، نام پوشه، محتوای فایل و تصاویر خودداری کنید.
  4. ساختار سرور را فاش نکنید - لینک‌ها و مسیرها به‌طور پیش‌فرض فهرست نمی‌شوند و قابل حدس نیستند.
  5. از سیستم‌های ایزوله استفاده کنید (مانند Tails، Qubes یا ماشین‌های مجازی).
  6. بر شبکه خود نظارت داشته باشید تا مطمئن شوید فقط از طریق آدرس .onion قابل دسترسی است.
  7. به‌روز بمانید - برای دریافت اصلاحات امنیتی و قابلیت‌های جدید، پروژه را دنبال کنید.

🔐 Добро пожаловать в TorServ

TorServ — это статический веб-сервер с усиленной безопасностью и нулевой настройкой, который автоматически запускается как скрытый сервис Tor. Он создан для анонимной публикации, особенно в условиях цензуры или высокого риска.

Распакуй → Запусти сервер → Готовый скрытый сервис за считанные секунды

Всё это за секунды — без сложной настройки и рисков использования небезопасного серверного ПО.
Бинарные пакеты доступны для Linux, Windows и Raspberry Pi

⚠️ Внимание: Ранняя стадия разработки

Проект находится на ранней стадии и активно развивается. Несмотря на реализованные меры безопасности, рассматривайте TorServ как экспериментальное программное обеспечение. Не полагайтесь на него в одиночку при выполнении задач с высоким уровнем риска. Используйте с крайней осторожностью в враждебных условиях.

🛡️ Основные функции безопасности

  • Только доступ через Tor — полностью без выхода в clearnet
  • Привязка только к локальному адресу (127.0.0.1)
  • Отсутствие логирования (доступ, ошибки, запросы)
  • Без аналитики и телеметрии
  • Удаление метаданных EXIF при запуске и во время работы
  • Удаление заголовков — User-Agent, Referer, ETag и др.
  • Случайная задержка ответа (от 50 до 200 мс)
  • Ротация ключей с аргументом командной строки --new-key
  • Выравнивание размера ответов (padding)
  • Запутывание URL и структуры путей файлов
  • Без индексации и автосписков каталогов
  • Сервер не запустится при наличии небезопасных типов файлов в директории public/
  • Динамическая проверка — небезопасные файлы не обслуживаются даже при добавлении во время работы

🧭 Как безопасно использовать

  1. Используйте только через браузер Tor. Никогда не открывайте в обычном браузере.
  2. Удаляйте метаданные из контента перед загрузкой — хотя TorServ удаляет большинство, лучше очистить всё вручную.
  3. Избегайте личной информации в именах файлов, папок, содержимом и изображениях.
  4. Не раскрывайте структуру сервера — ссылки и каталоги по умолчанию не индексируются и не угадываются.
  5. Используйте на изолированных машинах (рекомендуются Tails, Qubes или виртуальные машины).
  6. Следите за своей сетью, чтобы убедиться, что сервер доступен только по адресу .onion
  7. Обновляйтесь — следите за проектом, чтобы получать исправления и новые функции.
